# 动画

## 动画分帧

### 卡牌动画（card animation）

作用主体是卡牌或牌堆。

#### 摊牌动画（fan animation）

待机视图舞台，循环播放。

- 帧 1（0%）：牌堆静态叠放在舞台中央，所有牌完全重叠
- 帧 2（30%）：牌张沿弧线向左右两侧扇形展开，每张带轻微旋转
- 帧 3（55%）：扇形完全展开成对称弧形，短暂停顿
- 帧 4（85%）：反向收拢，牌张回到中央
- 帧 5（100%）：完全合叠，进入下一循环

#### 洗牌动画（shuffle animation）

占卜视图舞台。

- 帧 1（0%）：牌堆完整叠放在舞台中央
- 帧 2（25%）：牌堆从中央分裂为左右两叠
- 帧 3（55%）：左右两叠交错向中央汇聚（Riffle Shuffle 效果）
- 帧 4（85%）：交错完成，重新合为单叠
- 帧 5（100%）：单叠归位，准备进入切牌

#### 切牌动画（cut animation）

占卜视图舞台。

- 帧 1（0%）：牌堆居中静态
- 帧 2（30%）：上半叠向上抬起并向侧方偏移
- 帧 3（60%）：上半叠落到主叠侧方，形成两个分牌堆
- 帧 4（85%）：分牌堆稳定
- 帧 5（100%）：完成，进入抽牌前预备态

#### 抽牌动画（draw animation）

占卜视图舞台。

- 帧 1（0%）：分牌堆稳定
- 帧 2（30%）：单张牌从分牌堆顶部抽出，向上抬升
- 帧 3（70%）：牌移动到舞台中心，尺寸渐渐放大
- 帧 4（100%）：牌定格在中心位置，背面朝上——此即结果卡牌位置

#### 翻牌动画（flip animation）

占卜视图舞台。

- 帧 1（0%）：结果卡牌背面朝上静止
- 帧 2（30%）：沿垂直轴开始旋转，背面渐倾斜
- 帧 3（50%）：旋转至 90°，牌仅可见侧边线
- 帧 4（80%）：旋转至 180°，正面完整呈现
- 帧 5（100%）：正逆位标识浮现

### 几何动画（geometry animation）

作用主体是几何图形。

#### 兜底动画（fallback animation）

兜底视图舞台，循环播放，3D 立体优先。

- 静态元素：
  - 中央一颗静态发光恒星（球体）
  - 四条可见的椭圆轨道线，半径各异
  - 四颗 3D 立体几何行星：
    - 正四面体（tetrahedron）— 由原“等边三角”立体化
    - 平行六面体（parallelepiped）— 由原“平行四边形”立体化
    - 球体（sphere）— 由原“圆形”立体化
    - 八面体（octahedron）— 由原“菱形”立体化
- 运动：
  - 四颗行星沿各自椭圆轨道围绕恒星公转，半径与速度各异
  - 每颗行星以自身轴心持续自转
  - 缓慢、神秘、无限循环
- 不画任何行星之间的连线
- 实现技术：GSAP（CSS 3D transform + GSAP timeline + MotionPath 插件）

### 文字动画（text animation）

作用主体是文字。

#### 答案卡入场动画（answer rise-in animation）

答案卡（AnswerCard）内，结果区不再有逐字打字机，改用 CSS 分段 rise-in。

- 帧 1（0%）：原文、翻译、来源三段均不可见
- 帧 2（进行中）：三段自下而上分段浮现（rise-in），原文先、翻译次、来源末
- 帧 3（约 1.06s，prefers-reduced-motion 下立即）：三段全部呈现完毕，发出 typewriterComplete 完成信号，应用阶段由 reading 推进到 decision

## 视图过渡动画

视图切换时的过渡动画分两段：从待机视图到占卜视图，从占卜视图到结果视图（宽屏分栏 / 窄屏抽屉两个分支）。

### 第一段：待机视图 → 占卜视图

核心思路：舞台与牌堆是公共且连续的；标题与进度是上下交替的；牌堆从摊牌末态直接衔接进洗牌首态，不重建。

#### 阶段 1 — 等待摊牌收拢

1. 用户点击舞台触发后不立刻过渡。
2. 等当前摊牌动画播放到帧 5（100%，牌堆完全合叠），停止循环。
3. 等待时长 0~800ms，取决于点击瞬间摊牌动画的相位。

#### 阶段 2 — 元素交替 + 洗牌启动（同步进行，时长约 450ms）

1. 标题容器：向上平移 + 透明度 1 → 0 淡出。
2. 流程进度区：从舞台上方原标题位置（向上偏移约 16~24px 处）下移到目标位置 + 透明度 0 → 1 淡入。
3. 舞台上的牌堆（摊牌收拢的同一套牌）开始洗牌动画首帧（左右分裂）。
4. 缓动曲线：cubic-bezier(0.16, 1, 0.3, 1)（先快后慢，等价 GSAP 的 expo.out 或 power4.out）。

#### 阶段 3 — 占卜视图就位

1. 标题容器完全消失。
2. 流程进度区稳定在顶部。
3. 洗牌动画继续推进。

#### 关键设计要点

1. 舞台位置不变 — 待机视图与占卜视图共用同一舞台组件实例。
2. 牌堆是同一组卡牌实例，不销毁不重建，视觉连续。
3. 标题与进度的交替不打断舞台上的卡牌动画。

### 第二段：占卜视图 → 结果视图

触发时机：翻牌相位结束（翻牌动画帧 5：100% 完成，正逆位标识浮现完毕）。翻牌完成后停顿约 300ms，让用户看清正逆位标识，再启动结果视图入场。

#### 宽屏分支 → 结果分栏视图

##### 阶段 1 — 占卜舞台让位 + 分栏入场（同步进行，时长约 450ms）

1. 占卜视图宽度由 100% 收缩到 50%（左半屏），缓动 cubic-bezier(0.16, 1, 0.3, 1)。
2. 占卜舞台和结果卡牌随容器收缩，结果卡牌最终落位于左半屏中央。
3. 结果分栏视图从屏幕右外侧滑入，占据右半屏。
4. 操作区容器结构挂载但保持透明度 0，等待决策阶段再淡入。

##### 阶段 2 — 答案卡入场

1. 答案面板与其内的答案卡淡入。
2. 答案卡分段 rise-in：原文、翻译、来源自下而上依次浮现。
3. 此时为答案阶段（内部标识 reading），操作区不出现。

##### 阶段 3 — 决策阶段进入

1. 答案卡入场结束（约 1.06s，prefers-reduced-motion 下立即）发出 typewriterComplete 完成信号，进入决策阶段（decision）。
2. 操作区淡入（350ms，缓动 cubic-bezier(0.16, 1, 0.3, 1)），承载再占一次、回到首页按钮。
3. 异常态下操作区改为重试按钮。

#### 窄屏分支 → 结果抽屉视图

##### 阶段 1 — 抽屉从底部弹出（时长约 350ms）

1. 结果抽屉视图从屏幕底部边缘向上滑入，缓动 cubic-bezier(0.16, 1, 0.3, 1)。
2. 初始高度：抽屉顶端贴齐结果卡牌底部——这是抽屉的高度上限锚点。
3. 占卜视图保持原位，不收缩、不位移。
4. 操作区位于屏幕底部，独立于抽屉，容器结构挂载但保持透明度 0，等待决策阶段再淡入。

##### 阶段 2 — 答案卡入场

1. 答案面板与其内的答案卡淡入。
2. 答案卡分段 rise-in：原文、翻译、来源自下而上依次浮现。
3. 此时为答案阶段（内部标识 reading），操作区不出现。

##### 阶段 3 — 决策阶段进入

1. 答案卡入场结束（约 1.06s，prefers-reduced-motion 下立即）发出 typewriterComplete 完成信号，进入决策阶段（decision）。
2. 操作区淡入（350ms，缓动 cubic-bezier(0.16, 1, 0.3, 1)），承载再占一次、回到首页按钮。
3. 异常态下操作区改为重试按钮。

## 动效规范

1. 缓动语言：项目过渡统一采用 cubic-bezier(0.16, 1, 0.3, 1) 或同家族 cubic-bezier(0.32, 0.72, 0, 1)，先快后慢，末段柔和落定。
2. 时长基线：单段过渡动画 350~500ms；翻牌完成到结果视图入场之间留约 300ms 沉淀；答案卡分段 rise-in 入场约 1.06s，prefers-reduced-motion 下立即完成。
3. 操作区仅在决策阶段淡入，淡入时长 350ms，缓动 cubic-bezier(0.16, 1, 0.3, 1)；待机阶段、占卜阶段、翻牌相位、答案阶段（答案卡入场中）、兜底视图均无操作区。
4. 兜底视图标题文案“宇宙信号微弱，暂无法接通”作为唯一文字提示，不再叠加 toast 或通知。
